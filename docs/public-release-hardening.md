# ME3 Core Public Release Hardening

This checklist is the release gate for the first public ME3 Core baseline.

## D1 Migration Baseline

`apps/worker/migrations/0001_initial_public_schema.sql` is the initial public
D1 schema. It was generated by applying the private pre-release migration chain
through `0037_ai_gateway_external_routing_default.sql` to an empty database, then exporting only
public schema objects from `sqlite_schema`.

Before the first public tag, fresh installs should apply exactly this one SQL
migration. The previous private scaffold migration chain remains recoverable
from git history and release notes, but it is not part of the public install
path.

After the first public tag:

- Published migration files are immutable.
- Schema changes append a new numbered migration file.
- Do not rewrite, reorder, squash, or delete a migration that has shipped in a
  stable release.
- If a release includes Worker/D1 migrations since the previous stable release,
  set `migrationRequired: true` in `updates/stable.json`.
- Release notes must call out owner-visible migration or reset implications.

## Public Configuration Boundary

Public examples must not include real secrets, production Cloudflare resource
IDs, production ME3 Cloud routes, or hosted subscription billing config.

Allowed in public examples:

- Placeholder D1/R2 names and generated IDs.
- Owner-provided Worker secrets such as `SETUP_PASSWORD`, `JWT_SECRET`,
  `TOKEN_ENCRYPTION_KEY`, model provider keys, Telegram tokens, and Stripe keys.
- Optional commented examples for third-party or owner-operated services.

Not allowed in public defaults:

- Real `database_id`, account IDs, bucket IDs, webhook secrets, OAuth client
  secrets, API keys, JWT secrets, or encryption keys.
- Hosted ME3 subscription billing settings.
- Hosted-only ME3 Cloud routes enabled by default.
- Production custom domains or Worker route bindings owned by ME3 Cloud.

`ME3_SOCIAL_OAUTH_ORIGIN` is intentionally unset in the standalone Wrangler
example. A managed install or owner-operated bridge can configure it later.

## Security Gate

Before tagging a stable public release, verify:

- Secret scan: no committed `.dev.vars`, API keys, production Cloudflare IDs,
  webhook secrets, JWT secrets, encryption keys, OAuth private keys, or hosted
  subscription billing config.
- Auth/session: bootstrap, login, session hydration, logout, and protected
  owner routes are covered by Worker tests; cookies are `HttpOnly` and
  `SameSite=Lax`.
- CORS/origin posture: admin/login/API surfaces are not exposed from public
  root custom domains without the configured admin origin.
- Webhooks: provider webhooks use configured signing or token checks before
  acting on payloads.
- Abuse controls: public claim, login, reset, and setup paths have documented
  rate-limit or backoff coverage before launch.
- Least privilege: D1, R2, Durable Object, Workers AI, queue, email, and secret
  bindings are named explicitly in Wrangler examples; owners provide secrets
  through Worker secret storage, not config files.

## Performance Gate

Before tagging a stable public release, verify:

- `pnpm build` passes and reports no TypeScript or bundling errors.
- Fresh local D1 migration apply succeeds against an empty database.
- Worker deploy dry-run or build output is reviewed for bundle-size surprises.
- Hot install/update path runs `deploy:prepare` before D1 migrations and
  applies migrations before `wrangler deploy`.
- D1 query surfaces used during setup, login, public site serving, and update
  checks have indexes for owner/site/status/date lookups.
- Static assets are served through the configured Workers assets binding, with
  the Worker seeing requests first for custom-domain routing.
- Deploy-button and copied-repository update smoke tests are recorded in the
  relevant beads before the first public announcement.

## Validation Commands

Run these before closing the release-hardening bead:

```bash
pnpm --filter @me3-core/worker db:migrate:local
pnpm update:doctor
pnpm build
```

For release candidates, also run a real Cloudflare deploy/update smoke test from
a copied repository and record the result on the deploy/update beads.
